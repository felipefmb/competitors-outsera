    Nível 2 – Verbos HTTP e códigos de status
    Além de separar os recursos em URIs, agora se faz uso correto dos métodos HTTP:
    GET /filmes → listar
    POST /filmes → criar
    GET /filmes/{id} → buscar por id
    PUT /filmes/{id} → atualizar
    DELETE /filmes/{id} → remover
    Também deve retornar códigos de status HTTP adequados:
    200 OK, 201 Created, 404 Not Found, 400 Bad Request, etc.
    Esse nível é o que normalmente se espera de uma API RESTful “básica”.
